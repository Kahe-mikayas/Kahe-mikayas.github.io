Controle de Versão

 O controle de versão é um sistema feito com a finalidade de registrar mudanças feitas em um determinado arquivo. Ele é utilizado não somente na programação, mas também na edição de documentos de textos, imagens, vídeos e projetos dos mais diversos tipos. Existem diversas ferramentos que podem ser classificadas como locais, centralizadas e distribuídas. Os sistemas locais armazenam todas as alterações feitas em uma máquina, também armazena as diferenças permitindo que dados antigos sejam restaurados. Um exemplo disso é o RES (Revision Control System). Os sistemas centralizados permite que vários desenvolvedores sejam capazes de trabalharem juntos em um mesmo projeto, este sistema possui um servidor central que é responsável pelo armazenamento das versões dos arquivos. Exemplos deste sistema são o CVS e o Subversion. Os sistemas distribuídos funcionam de forma diferentes dos citados anteriormente, ao invés de tirar cópias das últimas versões, o repositório inteiro é copiado, para que assim, seja possível restaurar todo o servidor caso algo ocorra. Ele permite várias pessoas trabalhar em conjunto e também possibilita a criação de fluxos de trabalhos diferentes. Exemplos: git, mercural e Darcs.

 Para um melhor entendimento sobre o controle de versão, será utilizado o git como exemplo para a explicação. Apesar de possuir uma grande diversidade de comandos, os principais que serão utilizados são:

* git init: Inicializa um repositório.
* git add: Adiciona o repositório para a staged area.
* git commit: salva as alterações feitas.

E em caso o usuário esteja utilizando o git de forma remota, ele usará também.

* git clone (link do repositório): para obter uma cópia
* git push (link do repositório): para enviar os arquivos para o repositório principal.

 Uma forma organizada bastante utilizada no git é o uso de ramificações, no qual o usuário utiliza o comando git branch (nome), para criar uma cópia dos documentos em uma ramificação, git checkout, para troca de branch e por fim, git merge para mesclar tudo em um arquivo só. Isso também serve como uma forma de evitar erros que podem ocorrer por parte de ambos os programadores.

Teste

 O TDD, Desenvolvimento Orientado por Teste, traduzido do inglês, Teste Driven Development, é uma prática de Desenvolvimento de software utilizado que visa verificar se o código está funcionando de maneira correta no programa.  Vale lembrar que o teste é diferente de depuração. O teste é um processo de desenvolvimento como um todo. Eles são feitos por meio de ferramentas específicas que automatizam o processo e códigos, que possui a função de verificar se a função está gerando o valor esperado ou não. Os testes apenas verificam se o código está funcionando da maneira desejada, não o meio o utilizado de se chegar a ele.
 Já Depuração, diferente de teste, é o processo de se consertar algo que já sabe estar quebrado, nele é feito uma verificação passo-a-passo em busca de onde está localizado o problema e, caso seja necessário, serão feitas alterações no código do programa.
 O teste vai desde um nível de componente até um grande projeto, o qual possuirá grupos específicos para teste. Para cada um, existem diferentes técnicas e abordagens para as mais diversas situações.
 Para a criação de um programa é necessário ter 2 coisas em mente, a verificação, onde o software deverá funcionar da forma desejada, e validação, em que assegura que o software foi criado e atende aos requisitos do cliente.
 Como uma melhor forma de evitar erros, são feitos os testes alfa e beta, pois é praticamente impossível o desenvolvedor prever como o cliente utilizará o software. Os testes alfa são realizados na instalação do desenvolvedor por um grupo menor de pesssoas em um ambiente controlado. Já os testes beta são realizados nas instalações do usuário sem a presença de desenvolvedor, caso encontre um erro, o mesmo será relatado.

Framework

 O framework é uma estrutura de códigos que por objetivo facilitar a vida do desenvolvedor, trazendo novas funções, aumentando a produtivida e diminuindo o tempo gasto, além de trazer uma maior segurança para aqueles que irão utilizar o programa. Muitos acham que framework e biblioteca são a mesma coisa, mas na verdade são coisas bastante diferentes. Enquanto em uma biblioteca é o desenvolvedor que está no controle e escolhe quando a usar, no framework é ele quem escolhe quando chamar o código. Os framework possuem uma facilidade de se encaixar em qualquer contexto, o que por fim garante o bom funcionamento do código. Muitos dos frameworks utilizam a orientação a objetos com herança, polimorfismo e abstração.
Os frameworks possuem diversas vantagens, dentre elas:

* Código aberto, o que permite que qualquer um com conhecimentos sobre o código seja capaz de alterá-lo;
* Se encaixam em muitos contextos diferentes;
* Concede maior agilidade e segurança no código do programador;
* E permite maior legibilidade no código.

Todavia os frameworks também possuem as suas desvantagens, são elas:

* Dificuldade na sua configuração;
* Utilizar vários frameworks podem comprometer o código;
* Alguns são extensos e complexos, gerando uma barreira no entendimento do programado;
* Nem sempre são necessários, e por vezes podem acabar prejudicando no funcionamento do produto final.
